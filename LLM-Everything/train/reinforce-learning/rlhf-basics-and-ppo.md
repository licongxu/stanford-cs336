# RLHF基础

RLHF（Reinforcement Learning from Human Feedback）是将人类偏好引入大语言模型训练的核心技术范式。与传统的 SFT 不同，RLHF 不再追求"模仿人类书写的正确答案"，而是学习"生成人类更偏好的结果"。这一范式的完整流程可分为两个紧密衔接的阶段：

## 1 阶段一：训练奖励模型

Reward Model（奖励模型，简称 RM）是 RLHF 的"裁判系统"。其核心任务是为任意给定的（Prompt, Response）对输出一个标量奖励值，量化人类对该回复的偏好程度。

RM 通常基于预训练语言模型（如 LLaMA、GPT 系列）进行改造。具体实现上，在模型的最后一层隐藏状态之上添加一个**回归头**，将高维表示映射为单一标量值：

$$r_\theta(x, y) \in \mathbb{R}$$

其中 $$x$$ 表示输入提示（Prompt），$$y$$ 表示模型生成的回复（Response），$$\theta$$ 为 RM 的可训练参数。

架构选择上有两种主流方案：

* **共享参数方案**：复用预训练模型的全部参数，仅替换最后的输出层。优势在于保持强大的语义理解能力，劣势是训练成本较高。
* **冻结部分层方案**：冻结模型的底层参数（通常占 70%\~80%），仅微调顶层 Transformer 层和回归头。这能在保持 RM 判别能力的同时显著降低显存占用和训练时间。

### **1.1 损失函数：Pairwise Ranking Loss**

RM 的训练数据由成对比较构成：对于同一 Prompt $$x$$，人类标注者从两个候选回复 $$y_w$$（Win，被选中的）和 $$y_l$$（Lose，被拒绝的）中选出更优者。但神经网络需要数值才能优化。

Bradley-Terry 模型的核心洞察是：

> <mark style="color:$success;">如果 A 比 B 好，那么 A 的"质量分数"应该显著高于 B 的分数。</mark>

$$\mathcal{L} = -\log \sigma\bigl(r_\theta(x, y_w) - r_\theta(x, y_l)\bigr)$$

**符号解释：**

* $$y_w$$：Win（被人类选中的更好回复）
* $$y_l$$：Lose（被人类拒绝的较差回复）
* $$r_\theta(x, y)$$：奖励模型输出的标量分数（越高越好）
* $$\sigma$$：Sigmoid 函数，$$\sigma(z) = \frac{1}{1+e^{-z}}$$

对损失函数求导（链式法则），你会发现梯度为：

$$\frac{\partial \mathcal{L}}{\partial \theta} = \underbrace{\bigl[1 - \sigma(r_w - r_l)\bigr]}_{\text{误差项}} \cdot \underbrace{\bigl[\frac{\partial r_w}{\partial \theta} - \frac{\partial r_l}{\partial \theta}\bigr]}_{\text{奖励值对参数的梯度}}$$

**含义解读：**

1. **误差项** $$[1 - \sigma(r_w - r_l)]$$：
   * 当 $$r_w \gg r_l$$ 时，Sigmoid 接近 1，误差接近 0 → **梯度很小，几乎不再更新**
   * 当 $$r_w \approx r_l$$ 时，误差接近 0.5 → **梯度较大，大力调整**）
2. **调整方向**：
   * 增加获胜回复 $$y_w$$ 的奖励值（$$\frac{\partial r_w}{\partial \theta}$$ 为正）
   * 降低失败回复 $$y_l$$ 的奖励值（$$-\frac{\partial r_l}{\partial \theta}$$ 为负）

模型像一个跷跷板，一端抬起 Win，一端压低 Lose。

### **1.2 Margin** $$\delta$$ **的作用**

带 Margin 的版本： $$\mathcal{L} = -\log \sigma\bigl(r_w - r_l - \delta\bigr)$$

**为什么要加？**

想象两个场景：

* **场景 A**：$$r_w = 0.8, r_l = 0.7$$，差值 0.1
* **场景 B**：$$r_w = 0.8, r_l = -0.5$$，差值 1.3

没有 Margin 时，只要 $$r_w > r_l$$ 并且 $$r_w \ r_l$$ 的距离拉大，损失就会变小。

但问题在于，**微小的正差距（如 0.1）对应的损失（0.64）虽然比** $$r_w = r_l$$ 的 **0.69 略低，但这个优化信号较弱**，模型可能缺乏动力继续大幅拉开差距。

设置 $$\delta = 0.5$$ 后：

* 场景 A：$$0.8 - 0.7 - 0.5 = -0.4$$，Sigmoid(-0.4) ≈ 0.4，损失很大，模型必须继续拉大差距
* 场景 B：$$0.8 - (-0.5) - 0.5 = 0.8$$，Sigmoid(0.8) ≈ 0.69，损失较小

Margin 强迫模型不仅要知道"谁好"，还要明确"好多少"，避免对相近质量的回复给出暧昧的分数。

如果你熟悉分类任务，这个损失其实是**二元交叉熵的特例**：

标准的二元交叉熵： $$\mathcal{L}_{BCE} = -\bigl[y \cdot \log(p) + (1-y) \cdot \log(1-p)\bigr]$$

在我们的设定中：

* 标签 $$y = 1$$（始终假设 $$y_w$$ 应该赢）
* 概率 $$p = \sigma(r_w - r_l)$$

代入得到： $$\mathcal{L} = -\log \sigma(r_w - r_l) - 0 = -\log \sigma(r_w - r_l)$$

完全一样！所以你可以把它理解为：**在比较两个回复时，预测"左边比右边好"这个二元事件的概率，并最大化该概率**。

### 1.3 **In-batch Negative Sampling**（批次内负采样）

**In-batch Negative Sampling**（批次内负采样）是 RLHF 训练奖励模型时的一种**数据增强 trick**。它能在**不增加人工标注成本**的前提下，让模型见识更多的"负面例子"，从而提升判别能力和训练效率。

标准训练设定下，每条数据是： $$\text{(Prompt } x_i,\; \text{Win } y_i^w,\; \text{Lose } y_i^l)$$

对应损失： $$\mathcal{L}_i = -\log \sigma\bigl(r(x_i, y_i^w) - r(x_i, y_i^l)\bigr)$$

但是人类标注一对 $$(y^w, y^l)$$ 成本很高，而**每条数据只产生一次对比信号**。如果 batch size 是 64，一次迭代只能进行 64 次"裁判打分"练习。

**观察**：同一个 batch 内的其他样本，虽然来自不同 Prompt，但它们的 Lose 响应也是"不好的回复"。能不能**跨样本借用**这些负例？

假设 batch 中有 $$N$$ 个样本，记为：

* Prompts: $$x_1, x_2, ..., x_N$$
* 对应的获胜回复：$$y_1^w, y_2^w, ..., y_N^w$$
* 对应的失败回复：$$y_1^l, y_2^l, ..., y_N^l$$

**标准训练**：每个 $$y_i^w$$ 只与自己的 $$y_i^l$$ 对比

**In-batch 负采样**：让 $$y_i^w$$ 与**同 batch 内所有其他样本**的 $$y_j^l$$（其中 $$j \neq i$$）也进行对比

**新的损失函数**（以第 $i$ 个样本为例）：

$$\mathcal{L}_i = \underbrace{-\log \sigma(r(x_i, y_i^w) - r(x_i, y_i^l))}_{\text{原始监督信号}} \;-\; \lambda \sum_{j \neq i} \underbrace{\log \sigma(r(x_i, y_i^w) - r(x_j, y_j^l))}_{\text{In-batch 负样本信号}}$$

其中 $$\lambda$$ 是权重系数（通常设为 $$1/(N-1)$$ 或较小值如 0.1）。

这其实就是 **InfoNCE** 或 **对比学习** 的思想在自然语言处理中的应用：

* **锚点（Anchor）**：$$y_i^w$$（正样本）
* **正例（Positive）**：严格来说没有（或认为是自身）
* **负例（Negatives）**：$$y_i^l$$ 和 所有 $$y_{j\neq i}^l$$

## 2 阶段二：强化学习策略优化

获得训练好的 RM 后，进入 RLHF 的第二阶段：使用强化学习算法（这里以 PPO 为例，Proximal Policy Optimization）优化语言模型策略 $$\pi_\phi$$，使其生成能获得更高 RM 分数的回复。

**优化目标**

$$\max_{\phi} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\phi(y|x)} \left[ r_\theta(x, y) \right] - \beta \mathbb{D}_{\text{KL}} \left[ \pi_\phi(y|x) \| \pi_{\text{ref}}(y|x) \right]$$

目标函数包含两个关键项：

* **奖励最大化项**：鼓励策略生成高 RM 分数的回复
* **KL 散度约束项**：防止策略 $$\pi_\phi$$ 偏离参考模型 $$\pi_{\text{ref}}$$（通常是 SFT 模型）太远，维护语言模型的基础能力和输出分布稳定性

$$\beta$$ 为 KL 惩罚系数，是 RLHF 中最重要的超参数之一，通常需要在 0.01\~0.2 之间精细调优。

**Actor-Critic 架构**

PPO 算法采用经典的 Actor-Critic 结构：

* **Actor（策略模型）**：即被训练的语言模型，负责生成回复。其参数通过策略梯度更新，朝着高奖励方向优化。
* **Critic（价值模型）**：估计给定状态（已生成的 Token 序列）的期望累计回报，用于计算 Advantage（优势函数），降低梯度估计的方差。

### 2.1 Critic 模型

* **输入**：当前状态 $$s_t$$（即已生成的 Token 序列 $$[x, y_{<t}]$$，Prompt + 已生成的前 $$t-1$$ 个 token）
* **输出**：标量值 $$V_\phi(s_t)$$，表示"从当前状态出发，遵循当前策略，最终能获得的期望累计回报"

Critic 通常从 RM 或 SFT 模型初始化，其输出头被改造为价值输出。价值模型的损失函数为： $$\mathcal{L}_{\text{critic}} = \left( R_t - V_\psi(x, y_{<t}) \right)^2$$

其中 $$R_t$$ 为时刻 $$t$$ 的折扣回报。

**训练目标**：Critic 要尽可能准确地预测"从当前状态开始，最终能得多少分"。

#### 2.2 Generalized Advantage Estimation (GAE)

问题是：如何衡量"这一步有多好"？

我们需要计算 **Advantage（优势函数）** $$A_t$$，表示"在状态 $$s_t$$ 采取动作 $$a_t$$（生成第 $$t$$ 个 token）比平均水平好多少"。

有点复杂，我们把它当成在玩一个**二十一点（Blackjack）的游戏**，目标是让手牌点数接近21但不超过。

* **状态** $$s_t$$：你当前已经有的牌（比如"手里有16点"）
* **动作** $$a_t$$：你决定"要牌"（Hit）还是"停牌"（Stand）
* **完整回合**：你可能要很多次牌，最后庄家开牌，你**赢钱**或**输钱**（这就是最终的 Reward）

**问题**：如果你最后赢钱了，是**哪一次要牌**的功劳？

* 可能第一次要牌从12加到14很关键？
* 可能第二次从14加到16是冒险但正确的？
* 可能最后停在16点这个决策最明智？

这是一个 **Credit Assignment 的问题**：最终只有一个结果（赢/输），但你要给中间的**每一个动作**打分，告诉模型"这个动作对最终结果贡献了多少"。

Advantage 函数 $$A_t$$ 的核心定义： $$A_t = \underbrace{Q(s_t, a_t)}_{\text{采取这个动作的期望回报}} - \underbrace{V(s_t)}_{\text{这个状态下平均期望回报}}$$

**通俗解释**：

假设你手里有16点（状态 $$s_t$$）：

* **平均水平** $$V(s_t)$$：所有人在16点时，平均最终能赢多少钱（比如平均输0.5元）
* **采取动作"要牌"** $$Q(s_t, \text{要牌})$$：你在16点时执意要牌，最终能赢多少钱（比如平均赢0.3元）

那么： $$A_t = 0.3 - (-0.5) = 0.8$$

$$A_t > 0$$ **意味着**：在16点要牌这个决策**比平均水平好**，应该鼓励（下次在16点时更倾向要牌）。

现在的问题是：$$V(s_t)$$（平均水平）和 $$Q(s_t, a_t)$$（这个动作的价值）我们事先都不知道，必须通过采样来估计。

#### 方法一：蒙特卡洛（MC）——"打完看结果"

**做法**：真的把这局打完，看最后赢了多少钱。

$$A_t = \underbrace{R_t}_{\text{这一局最终实际赢的钱}} - \underbrace{V(s_t)}_{\text{模型估计的平均水平}}$$

例如：

* 你在16点选择要牌，最后拿到了21点，赢了100元，$$R_t = 100$$
* 模型估计 $$V(16\text{点}) = -0.5$$（平均输钱）
* 优势为：$$A_t = 100 - (-0.5) = 100.5$$

蒙特卡洛方法的**优点是它是一个无偏估计**。如果你打100万局，平均下来就是真实的期望回报。&#x20;

**缺点是方差极大**。这一局赢100元可能有多种原因，不仅仅是这一步的功劳。蒙特卡洛方法把后面所有的随机性都算到"第一步"的功劳/过错上，导致**信号噪声极大**。

#### 方法二：时序差分（TD）——"只看下一步"

不等到游戏结束，只看**下一步**怎么样。

$$A_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

其中：

* $$r_t$$：立即奖励（比如要牌后没爆，奖励0）
* $$V(s_{t+1})$$：进入新状态后的平均价值（比如到了18点，估计平均能赢0.1元）
* $$V(s_t)$$：原状态价值（16点，估计平均输0.5元）

**例子**：

* 你在16点要牌，拿到了2点，变成18点（没爆，$$r_t=0$$）
* 估计 $$V(18\text{点}) = 0.1$$，$$V(16\text{点}) = -0.5$$
* $$A_t = 0 + 0.9 \times 0.1 - (-0.5) = 0.59$$

时序差分的**优点是方差小**。只依赖下一步的状态，不关心后面庄家发什么牌，随机性小得多。&#x20;

**缺点是有偏估计**。如果 $$V(18\text{点})$$ 估计得不准（比如实际应该是-0.2），那这个偏差会传递给 $$A_t$$。

#### GAE 的解决思路

**GAE 的本质**：

> "结合蒙特卡洛和时序差分：既看一步（TD，稳定），也看两步、三步……直到看完整局（MC，准确）"

$$\hat{A}_t^{\text{GAE}} = \delta_t + (\gamma\lambda)\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + \cdots$$

其中 $$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$ 是 TD 误差。

**展开理解（以 3-step 为例）:**

$$\hat{A}_t = \underbrace{\delta_t}_{\text{1-step TD}} + \underbrace{(\gamma\lambda)\delta_{t+1}}_{\text{2-step}} + \underbrace{(\gamma\lambda)^2\delta_{t+2}}_{\text{3-step}} + \cdots$$

**直观解释**：

* $$\lambda = 0$$：只有第一项 $$\delta_t$$，纯 TD 方法
* $$\lambda = 1$$：无限累加，等价于 $$R_t - V(s_t)$$，纯 MC 方法
* $$\lambda = 0.95$$（常用值）：主要依赖实际观测的回报，但用价值函数削减部分方差

**回到打牌的例子**：

* TD 就像你刚要完牌，只看现在手牌好不好，不管后面怎么样（可能过于乐观或悲观）
* MC 就像你一定要打完才知道这步对不对（但可能这局赢钱纯属运气）
* GAE 就像你**既要考虑现在的手牌，也要看接下来几轮的走势，适当加权**，既不那么短视，也不被单局的运气带偏。
